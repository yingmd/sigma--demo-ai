# Figma 转前端代码

此技能指导将 Figma 设计稿转换为高质量前端代码的过程。

## 工作流程

1.  **分析设计需求**
    *   询问 Figma 设计链接、截图或详细描述（如果未提供）。
    *   识别布局结构（Grid, Flexbox）、颜色、排版和间距。
    *   识别可复用组件（按钮、输入框、卡片）。
    *   识别需要导出或引用的资源（图片、图标）。

2.  **确认技术栈**
    *   确认首选前端框架（React, Vue, Svelte, HTML/CSS）。
    *   确认样式方案（Tailwind CSS, CSS Modules, Styled Components, Sass）。
    *   确认 UI 库使用情况（如果有，例如 Shadcn UI, Material UI, Ant Design）。

3.  **组件架构**
    *   将设计拆分为组件层级结构。
    *   定义每个组件的 props 和 state。

4.  **实现步骤**
    *   **步骤 1：设置**：确保环境准备就绪（根据需要安装依赖）。
    *   **步骤 2：基础样式**：设置全局样式（颜色、字体）以匹配 Figma 变量。
    *   **步骤 3：原子/分子组件**：首先实现小型的可复用组件。
    *   **步骤 4：组织/页面**：将组件组装成更大的结构和页面。
    *   **步骤 5：响应式设计**：确保实现符合不同屏幕尺寸的设计要求。

5.  **代码质量检查**
    *   确保符合无障碍（A11y）标准（语义化 HTML, aria 标签）。
    *   验证实现是否与视觉设计匹配（尽可能像素完美）。
    *   优化图片和资源。

## 成功秘诀
*   使用语义化 HTML 标签（`<header>`, `<main>`, `<section>`, `<article>`）。
*   注意内边距、外边距和对齐方式。
*   将重复的样式提取为工具类或变量。
